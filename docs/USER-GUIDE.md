# Atlantic CRM — инструкция для пользователей

Руководство по работе в CRM Atlantic: от первого звонка клиента до закрытия сделки.

> **В CRM:** в боковом меню раздел **«Инструкция»** (`/instructions`) — краткая инструкция **для вашей роли** (менеджер, замерщик, цех и т.д.).  
> Этот файл — полное общее руководство для всех ролей.

## 1. Вход в систему

1. Откройте адрес CRM в браузере (например `http://ваш-сервер` или домен компании).
2. Введите **email** и **пароль**, выданные администратором.
3. После входа откроется главная страница для вашей роли (дашборд, замеры, производство или монтаж).

**Выход** — кнопка «Выйти» внизу бокового меню.

**На телефоне** — меню открывается кнопкой «гамбургер» вверху слева.

## 2. Роли и доступ

В CRM у каждого сотрудника своя роль. От роли зависит, какие разделы видны в меню.

| Роль | Кто это | Основные разделы |
|------|---------|------------------|
| **Администратор** | IT / руководитель | Всё, включая сотрудников и удаление заявок/заказов |
| **Менеджер** | Продажи | Заявки, клиенты, заказы, дашборд, производство (просмотр), монтаж (назначение) |
| **Экономист** | Финансы | Как менеджер, без редактирования части полей; акцент на оплатах и аналитике |
| **Замерщик** | Выезд на замер | Замеры, «Сегодня», календарь (только свои замеры) |
| **Монтажник** | Монтаж окон | Монтаж, «Сегодня», календарь (только свои монтажи) |
| **Мастер производства** | Общий цех | Очередь производства, все цеха на карточке |
| **Цех стекла / ПВХ / алюминия** | Узкий цех | Только свой цех в производстве |
| **Экономист** | — | Дашборд, аналитика, заказы (оплаты) |

> **Важно:** замерщик и монтажник в календаре видят **только свои** назначения, а не всю компанию.

## 3. Общая схема работы

Типичный путь сделки:

```text
Клиент → Заявка → Замер → КП / согласование → Заказ → Производство → Монтаж → Акт → Архив
```

### Этапы заявки (статусы)

| Статус | Что означает |
|--------|----------------|
| Новая | Заявка только создана |
| В работе | Менеджер ведёт клиента |
| Замер назначен | Назначена дата выезда замерщика |
| Замер выполнен | Замер сделан, есть данные/фото |
| КП отправлено | Клиенту отправили коммерческое предложение |
| Согласовано | Клиент согласился, можно создавать заказ |
| На производстве | Заказ отправлен в цех(а) |
| В производстве | Цех взял в работу |
| Готово к монтажу | Все цеха завершили изготовление |
| Монтаж назначен | Назначены дата и бригада |
| Смонтировано | Монтаж выполнен |
| Акт подписан | Подписан акт выполненных работ |
| Закрыта | Сделка в архиве |
| Отказ | Клиент отказался |

Статус меняется автоматически при ключевых действиях (создание заказа, отправка в производство, подписание акта) или вручную на карточке заявки.

## 4. Разделы CRM

### Продажи

| Раздел | Назначение |
|--------|------------|
| **Сегодня** | Замеры и монтажи на текущий день (удобно выездным сотрудникам) |
| **Календарь** | Месячная сетка: замеры и монтажи |
| **Дашборд** | Сводка: заявки, выручка, заказы, KPI, воронка |
| **Аналитика** | Отчёты и статистика |
| **Заявки** | Входящие обращения до заказа |
| **Клиенты** | База клиентов (имя, телефон, адрес) |

### Выполнение

| Раздел | Назначение |
|--------|------------|
| **Замеры** | Список и карточки замеров |
| **Заказы** | Сделки с спецификацией и оплатой |
| **Производство** | Очередь по цехам |
| **Монтаж** | Назначение и контроль монтажей |

### Управление

| Раздел | Назначение |
|--------|------------|
| **Задачи** | Внутренние поручения сотрудникам |
| **Гарантия** | Обращения по гарантии на заказы |
| **Архив** | Закрытые сделки |
| **Сотрудники** | Только администратор: учётные записи |

## 5. Заявки и клиенты

### Клиент

1. **Клиенты → Новый клиент** (или клиент создаётся при новой заявке).
2. Заполните: **имя**, **телефон** (обязательно), при необходимости email и **адрес**.
3. При создании с тем же телефоном CRM предупредит о **дубликате** — можно подтвердить создание или открыть существующего клиента.

### Заявка

1. **Заявки → Новая заявка** — выберите клиента, источник (звонок, сайт, Instagram и т.д.), описание.
2. На карточке заявки:
   - смена **статуса**;
   - назначение **замера**;
   - создание **заказа** (когда клиент согласился);
   - история всех изменений статуса.

### Удаление (только администратор)

На карточке заявки кнопка **«Удалить заявку»** — безвозвратно удаляет заявку, замеры и связанный заказ. Обычным сотрудникам эта кнопка не показывается.

## 6. Замеры

### Назначение замера

1. Откройте заявку → **Назначить замер** (или **Замеры → Новый** с привязкой к заявке).
2. Укажите:
   - **дату и время**;
   - **адрес объекта** (часто подставляется из карточки клиента);
   - **замерщика**.
3. Замерщик увидит его в **Замеры**, **Сегодня** и **Календаре**.

### Работа замерщика

1. Откройте карточку замера.
2. Отметьте **«В работу»**, затем по завершении — **«Выполнен»**.
3. Загрузите **фото и файлы** с объекта (с телефона можно с камеры).
4. При необходимости **перенесите** дату/адрес.

Адрес замера потом отображается на **карточке заказа**, если монтаж ещё не назначен.

## 7. Заказы

### Создание заказа

1. На заявке со статусом «Согласовано» (или раньше, по процессу компании) → **Создать заказ**.
2. Заполните **спецификацию**: позиции (тип изделия, размеры, цена), доп. работы, монтаж, срок изготовления.
3. После сохранения заявка получает статус **«Согласовано»**, появляется номер заказа вида **№00042**.

### Карточка заказа

На заказе видно:

- **Номер заказа**, клиент, телефон;
- **Адрес** (по приоритету: монтаж → последний замер → адрес клиента), ссылка на карту;
- спецификация и суммы;
- **оплаты** (предоплата, остаток) и долг;
- файлы (документы, наряды в цех);
- производство и монтаж;
- **чек-лист монтажа** (после назначения монтажа);
- подписание **акта** и отправка в архив.

### Оплата

В блоке «Оплата» менеджер/экономист фиксирует поступления. Статус оплаты: не оплачен / предоплата / оплачен.

### Отправка в производство

1. На заказе → **Отправить в производство**.
2. Выберите цеха: **Стекло**, **ПВХ**, **Алюминий** (можно несколько).
3. При необходимости приложите **наряды** (PDF/фото) для каждого цеха.
4. Заказ появится в разделе **Производство** у соответствующих сотрудников.

### Подписание акта и архив

1. Загрузите файл **акта выполненных работ** (фото или PDF).
2. Нажмите **«Подписать акт»**.
3. После подписания — **«Закрыть сделку и отправить в архив»**.

### Удаление заказа (только администратор)

Кнопка **«Удалить заказ»** на карточке — удаляет заказ и все связанные данные; **заявка остаётся**. Только роль **Администратор**.

### Список заказов

- Вкладки: **В работе**, **Акт подписан**, **Архив**.
- Фильтры по **этапу** и **оплате**.
- Поиск по клиенту, телефону, номеру заказа.
- Экспорт в Excel (менеджер, экономист, админ).

## 8. Производство

Раздел **Производство** — очередь заказов по статусам:

- **Ожидают запуска** — ещё не взяли в работу;
- **В работе** — хотя бы один цех начал;
- **Готово к монтажу** — все выбранные цеха нажали «Готово».

### Вкладки по цехам

Менеджер видит вкладки: **Все цеха**, **Стекло**, **ПВХ**, **Алюминий**.

Сотрудник цеха видит только **свои** заказы и только **свою** строку на карточке.

### Действия цеха

На карточке заказа в блоке цеха:

1. **«В работу»** — цех начал изготовление (первый цех переводит заказ в «В производстве»).
2. **«Готово»** — цех завершил свою часть.

Когда все назначенные цеха отметили «Готово», статус заказа становится **«Готово к монтажу»**.

### Наряды

Менеджер прикладывает наряды при отправке в производство. Цех видит **только свой** наряд (стекло / ПВХ / алюминий).

## 9. Монтаж

### Назначение

1. Когда заказ **готов к монтажу** → **Монтаж → Новый** или кнопка на карточке заказа.
2. Укажите: **дату и время**, **монтажника**, **адрес объекта**, примечания.
3. На карточке заказа появится блок «Монтаж» и **чек-лист** (пункты для отметки на объекте).

### Работа монтажника

1. **Монтаж** или **Сегодня** — список назначений.
2. На карточке заказа — отметить пункты чек-листа по мере выполнения.
3. После завершения — статус «Выполнен» (по процессу компании).

Адрес монтажа — **главный адрес** на карточке заказа (если монтаж уже назначен).

## 10. Задачи, гарантия, архив

### Задачи

- Создание задачи с исполнителем, сроком, привязкой к заявке.
- Статусы: ожидает → в работе → выполнена.
- Просроченные задачи подсвечиваются на дашборде.

### Гарантия

- Обращения по уже выполненным заказам.
- Статусы: открыто → в работе → решено.
- Привязка к номеру заказа и клиенту.

### Архив

Закрытые сделки (после подписания акта и архивации). Поиск по клиенту и фильтр по статусу.

---

## 11. Поиск и уведомления

### Глобальный поиск

В верхней части экрана (на десктопе) — поиск по:

- клиентам;
- заявкам;
- заказам (в том числе по **номеру**).

На страницах **Заказы**, **Заявки**, **Клиенты**, **Архив** — своё поле поиска в списке (глобальный поиск там скрыт, чтобы не дублировать).

### Уведомления

Колокольчик в боковом меню — уведомления CRM (например, о новом заказе). Непрочитанные отмечены счётчиком.

### Push-уведомления в браузере

Кнопка **«Включить уведомления»** внизу меню — подписка на push (если браузер разрешит). Удобно для производства и менеджеров.

## 12. Файлы и фото

- Загрузка с компьютера или **с камеры** телефона.
- Фото **сжимаются** перед отправкой (если файл большой) — быстрее грузится на слабом интернете.
- Удаление файла (крестик) — файл убирается из CRM и из хранилища.
- Поддерживаются документы и изображения (акты, наряды, фото с объекта).

Если загрузка не работает — сообщите администратору (часто нужна настройка сервера файлов).

## 13. Частые вопросы

**Не вижу нужный раздел в меню**  
У вашей роли нет доступа. Обратитесь к администратору для смены роли.

**Не могу удалить заявку или заказ**  
Удаление доступно только **администратору**. Менеджер закрывает сделку через **акт → архив**.

**На заказе неверный адрес**  
Проверьте адрес **монтажа**, последнего **замера** и карточки **клиента** — на заказе показывается адрес монтажа, если он назначен, иначе замер, иначе клиент.

**Замерщик видит чужие замеры в календаре**  
Не должен — в календаре только свои. Если видит чужие, сообщите администратору (обновление системы).

**Два заказа на одну заявку**  
На одну заявку — один заказ. Создавайте новую заявку для повторной сделки.

**Как установить CRM на телефон**  
В Chrome/Safari: меню браузера → «На главный экран» / «Добавить на экран». Откроется как приложение (PWA).

**Забыл пароль**  
Только администратор может сбросить пароль в разделе **Сотрудники**.
